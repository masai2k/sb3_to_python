Convert `.sb3` files (Scratch / PenguinMod projects) into Python code, and optionally auto-debug the generated `.py` file using a C++ debugger.

---

## Features

- Convert `.sb3` files into `.py`
- Generate a debugged Python file automatically
- GUI launcher with **Convert** and **Debug** buttons
- Terminal usage
- C++ debugger generation for syntax-fix passes

---

## Files

- `convertitore.py` → converts `.sb3` into `.py`
- `debug.py` → takes a `.py` file, generates `sb3_debugger.cpp`, compiles it, and creates `yourfiledebuggato.py`
- `home.py` → opens a GUI with **Convert** and **Debug** buttons

---

## Requirements

- Python 3
- Tkinter
- `g++` or `clang++` for the C++ debugger

---

## Installing Requirements

### Python

Check that Python 3 is installed:

    python3 --version

If needed, install it from your system package manager or from the official Python website.

---

### Tkinter

#### Ubuntu / Debian

    sudo apt update
    sudo apt install python3-tk

#### Fedora

    sudo dnf install python3-tkinter

#### Arch Linux

    sudo pacman -S tk

#### macOS

Tkinter is usually included with the official Python installer from python.org.

To verify:

    python3 -m tkinter

If a small window opens, Tkinter is working.

---

### g++ or clang++

You only need one of them.

#### Ubuntu / Debian

Install `g++`:

    sudo apt update
    sudo apt install g++

Or install `clang++`:

    sudo apt update
    sudo apt install clang

#### Fedora

Install `g++`:

    sudo dnf install gcc-c++

Or install `clang++`:

    sudo dnf install clang

#### Arch Linux

Install `g++`:

    sudo pacman -S gcc

Or install `clang++`:

    sudo pacman -S clang

#### macOS

Install Apple Command Line Tools:

    xcode-select --install

This provides `clang++`.

To verify:

    clang++ --version

## GUI Usage

Run the GUI directly from GitHub:

    curl -fsSL https://raw.githubusercontent.com/masai2k/sb3_to_python/main/home.py | python3 -

This opens a window with two buttons:

- Convert
- Debug

### Convert button

When you click **Convert**:

- it asks you to choose an `.sb3` file
- it runs `convertitore.py`
- it generates:
  - `yourfile.py`
  - `yourfiledebuggato.py` (if the C++ debug step succeeds)

### Debug button

When you click **Debug**:

- it asks you to choose a `.py` file
- it runs `debug.py`
- it generates:
  - `yourfiledebuggato.py`

---

## Terminal Usage

### Convert an SB3 file

    curl -fsSL https://raw.githubusercontent.com/masai2k/sb3_to_python/main/convertitore.py | python3 - yourfile.sb3

This generates:

- `yourfile.py`
- `yourfiledebuggato.py`

### Debug a Python file

    curl -fsSL https://raw.githubusercontent.com/masai2k/sb3_to_python/main/debug.py | python3 - yourfile.py

This generates:

- `yourfiledebuggato.py`

---

## Output Files

### After conversion

If the input file is:

    yourfile.sb3

The generated files will be:

- `yourfile.py`
- `sb3_debugger.cpp`
- `sb3_debugger`
- `yourfiledebuggato.py`

### After debug only

If the input file is:

    yourfile.py

The generated files will be:

- `sb3_debugger.cpp`
- `sb3_debugger`
- `yourfiledebuggato.py`

---

## How the pipeline works

### Convert pipeline

    yourfile.sb3
       ↓
    convertitore.py
       ↓
    yourfile.py
       ↓
    sb3_debugger.cpp
       ↓
    compiled C++ debugger
       ↓
    yourfiledebuggato.py

### Debug pipeline

    yourfile.py
       ↓
    debug.py
       ↓
    sb3_debugger.cpp
       ↓
    compiled C++ debugger
       ↓
    yourfiledebuggato.py

---

## Notes

- `yourfiledebuggato.py` is generated by the C++ debugger, not directly by Python
- the debugger focuses on common syntax issues
- unsupported Scratch / PenguinMod blocks may still appear as fallback comments in the generated Python
- if no C++ compiler is installed, the debug stage cannot run
